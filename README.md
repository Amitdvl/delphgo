# Delphgo
<img width="1536" height="1024" alt="delphgo" src="https://github.com/user-attachments/assets/ef232956-b7dd-40d7-8d59-e814388c0d09" />

A read-only Go CLI that fetches and summarizes [Chief Delphi](https://www.chiefdelphi.com) forum activity. Uses the public RSS feed as the primary ingestion path, with optional deep scraping of individual topic pages. Summaries are generated by **Claude Sonnet 4.6** by default, with a rule-based fallback if no API key is set.

## Install

```bash
git clone https://github.com/Amitdvl/delphgo.git
cd delphgo
go build -o delphgo .
```

Requires Go 1.21+.

## Setup

Set your Anthropic API key to enable AI-powered summaries:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
```

Without it, delphgo automatically falls back to rule-based summaries. No other configuration is required.

## Usage

```
delphgo [flags]
```

### Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--limit N` | `30` | Max number of topics to process |
| `--category NAME` | | Filter by category (e.g. `Technical`, `Programming`, `Robot Showcase`) |
| `--keyword WORD` | | Filter by keyword in title, content, or author |
| `--tags TAG1,TAG2` | | Filter by comma-separated tags |
| `--since DURATION` | | Only topics from the last duration (e.g. `24h`, `72h`, `168h`) |
| `--deep` | `false` | Fetch individual topic pages to extract replies — feeds richer data to the AI summary |
| `--workers N` | `5` | Number of concurrent scrapers in `--deep` mode |
| `--no-llm` | `false` | Use rule-based summary instead of Claude Sonnet 4.6 |
| `--output FILE` | stdout | Write Markdown output to a file |
| `--feed-url URL` | CD latest RSS | Override the RSS feed URL |

### Examples

```bash
# AI summary of latest topics (requires ANTHROPIC_API_KEY)
./delphgo

# Deep scrape replies, then summarize with AI
./delphgo --deep --limit 15

# Filter to a category, AI summary
./delphgo --category Programming

# Search for a keyword
./delphgo --keyword swerve

# Last 24 hours, write to file
./delphgo --since 24h --output summary.md

# No API key needed — rule-based summary
./delphgo --no-llm

# Category-specific RSS feed with AI summary
./delphgo --feed-url https://www.chiefdelphi.com/c/technical/43.rss --deep
```

## Output

delphgo generates a Markdown document with five sections:

**Main Themes** — What subjects dominate the discussion, grouped by theme.

**Important Threads** — The most notable or high-engagement threads with author, link, and description. In `--deep` mode, notable replies are quoted inline.

**Actionable Takeaways** — Help requests, announcements, rule clarifications, and events that teams should know about.

**Useful Links** — Curated list of topic links with category labels.

**Repeated Opinions & Sentiment** — Recurring views, concerns, or enthusiasm the community is expressing.

Progress and warnings are written to stderr; the Markdown summary goes to stdout (or `--output`).

## How it works

1. Fetches the Chief Delphi RSS feed (`/latest.rss` by default)
2. Parses the XML to extract topic title, author, category, timestamp, and first-post excerpt
3. Applies any active filters (category, keyword, tags, time window)
4. If `--deep` is set, concurrently fetches each topic page and parses Discourse's crawler-friendly HTML to extract reply authors and content
5. Sends the structured topic data to **Claude Sonnet 4.6** to generate a high-quality summary (or falls back to rule-based if `ANTHROPIC_API_KEY` is not set or `--no-llm` is passed)

No login or credentials required for data ingestion — everything comes from public pages.
